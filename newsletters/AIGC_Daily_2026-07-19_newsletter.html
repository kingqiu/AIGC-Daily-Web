MCP vs A2A vs ACP：AI智能体之间如何实现通信

这篇 ByteByteGo 文章切入了一个正在悄悄撕裂 agent 生态的问题：AI agent 之间、以及 agent 与工具之间，到底该怎么"说话"？文章横向对比了目前最热的三套协议——Anthropic 的 MCP（Model Context Protocol）、Google 的 A2A（Agent-to-Agent），以及 IBM 的 ACP（Agent Communication Protocol）。

最有价值的部分是三者在设计哲学上的根本分歧。MCP 把工具和数据源视为一等公民，agent 通过标准化的 client-server 模型去发现和调用它们；A2A 更激进，主张 agent 之间直接做能力广告和任务协商，走 peer-to-peer 路线；而 ACP 来自企业场景，天生把治理、认证和审计日志写进协议层。文章对三者在互操作性、安全性和开发者体验上的取舍做了清晰拆解，没有流于表面罗列。
